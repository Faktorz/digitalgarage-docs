Image Registry

  • Overview
  • Integrated the Digital Garage Registry
  • Third Party Registries
    • Authentication

Overview

the Digital Garage can utilize any server implementing the Docker registry API as a source of images, including the canonical Docker Hub, private registries run by third parties, and the integrated the Digital Garage registry.

Integrated the Digital Garage Registry

the Digital Garage provides an integrated Docker registry that adds the ability to provision new image repositories on the fly. This allows users to automatically have a place for their builds to push the resulting images.

Whenever a new image is pushed to the integrated registry, the registry notifies the Digital Garage about the new image, passing along all the information about it, such as the namespace, name, and image metadata. Different pieces of the Digital Garage react to new images, creating new builds and deployments.

Third Party Registries

the Digital Garage can create containers using images from third party registries, but it is unlikely that these registries offer the same image notification support as the integrated the Digital Garage registry. In this situation the Digital Garage will fetch tags from the remote registry upon imagestream creation. Refreshing the fetched tags is as simple as running oc import-image <stream>. When new images are detected, the previously-described build and deployment reactions occur.

Authentication

the Digital Garage can communicate with registries to access private image repositories using credentials supplied by the user. This allows the Digital Garage to push and pull images to and from private repositories. The Authentication topic has more information.
