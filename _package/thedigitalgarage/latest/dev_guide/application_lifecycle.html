Application Life Cycle Examples

  • Overview
  • Developing on Digital Garage
  • Developing Then Deploying on Digital Garage

Overview

As a PaaS, Digital Garage is designed for building and deploying applications. Depending on how much you want to involve Digital Garage in the development process, you can choose to develop on Digital Garage and use it to continuously develop an application, or you can deploy a fully developed application onto an Digital Garage instance.

Developing on Digital Garage

You can develop your application on Digital Garage directly. Use the following process if you plan to use Digital Garage as a method to build and deploy your application:

Initial Planning

  • What does your application do?

  • What programming language will it be developed in?

Access to Digital Garage

  • Digital Garage should be installed by this point, either by yourself or an administrator within your organization.

Develop

  • Using your editor/IDE of choice, create a basic skeleton of an application. It should be developed enough to tell Digital Garage what kind of application it is.

  • Push the code to your Git repository.

Generate

  • Create a basic application using the new-app command. Digital Garage generates build and deployment configurations.

Manage

  • Start developing your application code.

  • Ensure your application builds successfully.

  • Continue to locally develop and polish your code.

  • Push your code to a Git repository.

  • Is any extra configuration needed? Explore the Developer Guide for more options.

Verify

  • You can verify your application in a number of ways. You can push your changes to your application’s Git repository, and use Digital Garage to rebuild and redeploy your application. Alternatively, you can hot deploy using rsync to synchronize your code changes into a running pod.

Developing Then Deploying on Digital Garage

Another possible application life cycle is to develop locally, then use Digital Garage to deploy your fully developed application. Use the following process if you plan to have application code already, then want to build and deploy onto an Digital Garage installation when completed:

Initial Planning

  • What does your application do?

  • What programming language will it be developed in?

Develop

  • Develop your application code using your editor/IDE of choice.

  • Build and test your application code locally.

  • Push your code to a Git repository.

Access to Digital Garage

  • Digital Garage should be installed by this point, either by yourself or an administrator within your organization.

Generate

  • Create a basic application using the new-app command. Digital Garage generates build and deployment configurations.

Verify

  • Ensure that the application that you have built and deployed in the above Generate step is successfully running on Digital Garage.

Manage

  • Continue to develop your application code until you are happy with the results.

  • Rebuild your application in Digital Garage to accept any newly pushed code.

  • Is any extra configuration needed? Explore the Developer Guide for more options.
